# Batching 深层推导

> 从"多请求一起算"往下推，推到不能再分

---

## 表层问题

**输入**：请求集合 [req1, req2, req3]
**输出**：批量计算结果
**问题**：如何让多个请求一起算？

---

## 第一层推导

```
    [请求 → 批]
          │
    ┌─────┴─────┐
    │           │
[静态批] [动态批]
    │           │
    ▼           ▼
[Fixed Size] [Marquez]
    │           │
    └─────┬─────┘
          │
    [Continuous Batching]
```

| 层级 | 问题 | 解决 | 概念 |
|------|------|------|------|
| 表层 | 多请求一起算？ | 批处理 | Batching |
| 底层 | 批大小固定？ | 静态批 | Static Batch |
| 底层 | 动态加入？ | 动态批 | Continuous Batching |
| 底层 | 何时换出 | 演化算法 | Policy |

---

## 第二层推导：静态 vs 动态

### 2.1 静态批处理

```
[Static]
    │
    ├── 固定大小 (如 batch=4)
    │       │
    │       所有请求一起算完
    │
    ├── 问题：有人慢怎么办？
    │       解决：padding到同长度
    │
    └── 问题：有人先完？
            等待，浪费
```

### 2.2 动态批处理

```
[Continuous]
    │
    ├── 有人完成就加入新的
    │       │
    │       问题：何时加入？
    │               动态检测
    │
    └── 关键算法：
            └── 队列演化
```

### 2.3 PagedAttention 批处理

```
[PagedAttn + Batch]
    │
    ├── 优势：更灵活的批
    │       │
    │       不需要等长
    │
    └── 核心：
            动态块分配
```

---

## 第三层推导：Policy

### 3.1 加入 Policy

```
[Add Policy]
    │
    ├── 问题：何时加新请求？
    │       解决：GPU有空闲
    │
    ├── 问题：加多少？
    │       解决：填满为止
    │
    └── 问题：优先级？
            优先级队列
```

### 3.2 移除 Policy

```
[Remove Policy]
    │
    ├── 问题：何时移除？
    │       解决：终止token
    │
    ├── 问题：超时？
    │       设置超时
    │
    └── 问题：OOM？
            强制移除
```

### 3.3 演化算法

```
[Evolution]
    │
    ├── Max num tokens
    │       限制总token
    │
    ├── Max running sequences
    │       限制并
    │
    └── Preemption
            允许抢占
```

Batching

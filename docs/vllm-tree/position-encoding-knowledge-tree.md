# 位置编码知识树

> 从"位置编码到底在学什么？"出发的完整推导链

---

## 来时路

```
计算环节 → Transformer → Attention → 位置信息丢失 → 位置编码
```

从 Transformer 的置换不变性问题出发，推导位置编码的必要性和各种方案。

---

## 根问题：位置编码到底在学什么？

### 第一问：位置信息包含哪些内容？

| 层次 | 含义 | 例子 |
|------|------|------|
| 相对距离 | token A 和 token B 之间隔了多少个词 | "猫"和"老鼠"距离 2 |
| 绝对位置 | token 在序列中的绝对索引 | 它是第 5 个词 |
| 结构边界 | 是否跨句子、段落 | 上一句的结尾与下一句的开头 |

**语言中最重要的通常是相对距离**，因为语法依赖（主谓、动宾）和共指（代词-实体）都基于相对距离。

---

### 第二问：这些位置信息如何"注入"模型？

有两种根本不同的"注入"方式：

**方式一：加在输入向量上（Absolute PE）**
```
X = Token_Embed + Position_Signal(pos)
```
这相当于告诉模型："这个词不仅有自己的语义，还有一个位置标识符（颜色、形状）叠加在上面。"

模型需要在后续的 QK^T 计算中自行分离和解读这个混合信号。

**方式二：直接修改注意力分数（Relative Bias）**
```
Score = Q·K^T + Bias(relative_distance)
```
这相当于直接告诉注意力机制："距离越远，扣分越多" 或 "特定距离对，加上特定偏差"。

模型不需要从输入特征中解析位置，位置直接作用在注意力这个"连接选择器"上。

---

### 第三问：位置编码的参数/公式在优化什么？

#### 3.1 可学习绝对位置编码

**参数**：一个矩阵 W_pos ∈ R^{max_len × d_model}，每一行是一个位置的嵌入。

**学习目标**：
通过反向传播，让每个位置的嵌入向量成为帮助 LM 任务损失减小的最优"位置角色"向量。

它不需要显式遵守任何函数规律，只需在训练数据中"好用"。

**实际学到了什么**（实验观察）：
- 邻近位置向量相似：因为自然语言中相邻 token 经常处于同一短语，梯度对邻近位置给出了相似的更新，于是它们的向量在空间中靠近（高余弦相似度）
- 距离的间接编码：通过位置-位置点积 (W_q·P_i)·(W_k·P_j)，模型可以学会某个 (i,j) 对组合产生特定的注意力偏置。但这种"学会"是对每个 (i,j) 单独记忆，没有抽象出"距离 k"的函数

**本质**：一张离散的、每个位置独立优化的"查找表"。

---

#### 3.2 公式化绝对位置编码（Sinusoidal）

**公式**：
```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

**学习目标**（非参数）：不需要梯度学习，它的设计目标是通过数学结构满足三个理想特性：

1. **唯一性**：不同位置的向量不同（多频率组合给出二进制编码式唯一性）
2. **相对距离可恢复**：两个位置的 PE 点积只与相对距离 k 有关，这在数学上成立
3. **平滑性**：相邻位置向量在高频维度变化大，低频维度相似，形成多尺度表征

**它学会了什么？**
它什么都没"学"（参数固定），但编码了相对距离的连续函数。模型通过训练 W_q、W_k 学会解读这些固定信号，实际上是在优化从这些正弦波中提取相对距离的能力。

---

#### 3.3 相对位置偏置（ALiBi）

**"参数"**：每头一个固定斜率 m。

**学习目标**：不学习位置表示本身，而是直接告诉注意力：距离 |i-j| 的对数影响是负线性的。

**学到的本质**：一个极强归纳偏置——距离衰减的单调线性规律。它没有学习的余地（除了 m 的大小顺序预设），模型被迫适应这个规律，而这个规律恰好适用于任意长度。

---

#### 3.4 旋转位置编码（RoPE）

**公式**：将 Q、K 按维度对旋转 pos·θ_i。

**学习目标**：
与 Sinusoidal 类似，RoPE 通过数学构造让 Q·K 的内积变为只依赖相对距离 m-n 的函数。

但 RoPE 通过旋转作用于 Q、K，而不是加在 X 上，所以位置信号直接参与注意力计算，没有内容和位置交叉项的干扰。

旋转频率 θ_i 也是固定的，但不同于 Sinusoidal 直接加到输入端，RoPE 使相对距离成为内积的乘性调制。

**真正在学什么？**
RoPE 没有可学习的位置参数，它学的是模型对"在不同相对距离下，注意力应该有多强"这个映射的权重（通过 W_q、W_k 适应旋转后的内积分布）。由于这个映射是相对距离的平滑函数，外推时可以通过缩放频率来适应更长距离。

---

### 第四问：评价位置编码好坏的隐式"损失函数"

虽然位置编码本身不一定由梯度直接优化，但所有位置编码方案都接受同一个元评价标准：

| 标准 | 含义 | 高下立判 |
|------|------|----------|
| 可区分性 | 不同位置应产生不同的注意力行为 | 所有方案基本达成 |
| 平滑性 | 邻近位置应比远距离位置更相似 | 可学习（间接有）、Sinusoidal、ALiBi、RoPE 都满足 |
| 外推性 | 未见过的位置要能合理推理 | ALiBi > RoPE(NTK) > Sinusoidal >> 可学习 |
| 与内容的可分离性 | 位置不应淹没内容，尤其在远距离 | RoPE 最优（乘法解耦），ALiBi 次之（加法偏置），混合加法最差 |

---

## 四种方案详细知识树

### 1. 可学习绝对位置编码

```
[根] 可学习绝对位置编码：怎么让模型"背下"每个位置？
│
├── Q1: 这个参数矩阵长什么样？
│   ├── Q1.1: 形状是谁决定的？
│   │        A: 行数 = 预设最大序列长度 max_len（如 512）
│   │           列数 = 隐藏维度 d_model（如 768）
│   │           所以矩阵 W_pos ∈ R^{max_len × d_model}
│   │
│   ├── Q1.2: 初始化用什么值？
│   │        A: 通常用 N(0, 0.02) 之类的随机正态分布
│   │           也有用均匀分布，但必须随机，不能全为0
│   │           否则对称位置没有差异，注意力无法打破
│   │
│   └── Q1.3: 它和 Token Embedding 是同一个矩阵吗？
│            A: 不是，完全独立。Token Embedding 大小是 [vocab_size, d_model]
│               位置 Embedding 大小是 [max_len, d_model]
│               二者分别查表，然后相加
│
├── Q2: 加法具体是怎么"加"进去的？
│   ├── Q2.1: 加法时的形状广播（Batch=1, seq_len=3）
│   │        Input token IDs: [15, 208, 99]     -> Token Embed -> [3, 768]
│   │        Position indices: [0, 1, 2]        -> Position Embed -> [3, 768]
│   │        X = Token_Embed + Position_Embed   (逐元素相加，形状不变)
│   │
│   ├── Q2.2: 超出 max_len 的位置怎么处理？
│   │        A: 训练时通常直接截断到 max_len；推理时如果序列过长，
│   │           要么截断，要么使用某种外推 hack（但原方案不支持）
│   │
│   └── Q2.3: 每一层 Transformer 都加一次位置编码吗？
│            A: 原始设计只在输入层加一次。
│               后续层通过残差连接传递位置信号，不再重复加。
│               有些变体每层都加，但可学习绝对位置的主流只加在输入。
│
├── Q3: 为什么必须用加法，而不是拼接？
│   ├── Q3.1: 拼接会怎样？
│   │        如果拼接: X = [Token_Embed; Position_Embed] 形状 [3, 768+768=1536]
│   │        则后续 W_q, W_k, W_v 形状必须变为 [1536, 768]
│   │        参数量增加一半，且位置信息与语义信息严格隔离
│   │        实验发现没必要，加法足以让模型自行交织两者
│   │
│   ├── Q3.2: 加法后，语义和位置不会混淆吗？
│   │        A: 正是我们需要的"软交织"。
│   │           模型通过训练，会学会在 Attention 点积中提取
│   │           "内容-内容"、"位置-位置"、"内容-位置"等交叉项
│   │           加法恰好在不增加维度的情况下提供了这些交叉项
│   │
│   └── Q3.3: 那乘法（如 RoPE）比加法强在哪里？
│            A: 加法是平移，乘法是旋转。
│               平移产生的点积会混杂绝对位置项；
│               旋转让最终点积只依赖相对距离。
│               但可学习位置编码的"加法"连平移都不是精确平动，
│               因为 W_pos 每行独立，无法保证平移等距性，
│               所以外推困难。
│
├── Q4: 训练结束后，位置向量学到了什么？模型如何利用它们？
│   ├── Q4.1: 相邻位置的向量真的更相似吗？
│   │        A: 实验观察：pos i 和 i+1 的余弦相似度通常较高
│   │           但这不是通过平滑性约束训练的，而是自然出现的
│   │           因为短距离相对关系在语料中出现频率更高，
│   │           梯度推动了邻近位置向量靠近
│   │
│   ├── Q4.2: 位置向量会形成像正弦那样的波形吗？
│   │        A: 通常不会形成严格的三角函数波，而是更嘈杂的、层次化的模式
│   │           低维可能编码宽范围的距离，高维编码细粒度局部差异
│   │           但与 Sinusoidal 不同，没有明确的周期性和外推能力
│   │
│   ├── Q4.3: 注意力计算中怎么拆解位置的作用？
│   │        对于 Q = (X_i + P_i)W_q, K = (X_j + P_j)W_k
│   │        点积展开：
│   │        Q·K = (X_i W_q)·(X_j W_k)     ← 内容-内容
│   │            + (X_i W_q)·(P_j W_k)     ← 内容-位置
│   │            + (P_i W_q)·(X_j W_k)     ← 位置-内容
│   │            + (P_i W_q)·(P_j W_k)     ← 位置-位置
│   │        这四项让模型可以同时利用绝对和相对位置信息
│   │
│   └── Q4.4: 它能学到"相对距离"吗？
│            A: 间接能。通过位置-位置项 (P_i'·P_j')，
│               模型可以记住某些 i-j 距离组合对应的分数，
│               但这种记忆是对每个 (i,j) 对的独立参数组合，
│               不是显式的相对距离函数，无法外推。
│
└── Q5: 外推彻底失败的原因——深层数理分析
    ├── Q5.1: 工程上常见的越界 hack 有哪些？后果是什么？
    │        - 截断 (clamp(pos, 0, max_len-1))：
    │          长序列尾部多个 token 共享同一个位置向量，
    │          导致它们位置区分度丧失，模型视为同一位置
    │        - 用全零向量或随机向量：
    │          输入分布剧变，注意力分数出现异常极大或极小值
    │          模型行为不可控，通常生成乱码
    │
    ├── Q5.2: 为什么模型把它当成"绝对陌生的变量"？
    │        A: 模型是一个巨大的函数逼近器。训练期间，它只在
    │           {0,1,...,L-1} 这些离散输入上见过损失函数的梯度。
    │           对位置 L，梯度从未影响到任何 W_pos[L] 行（因为不存在）。
    │           因此模型在 L 上的行为完全未定义。
    │
    ├── Q5.3: 如果把最后一个已知位置向量复制给新位置呢？
    │        A: 位置 L-1 的向量是专门为"最远端"优化的表示，
    │           适用于 L-1 及其附近的相对距离模式。
    │           把 L-1 复制给 L, L+1,... 会消灭长程衰减趋势，
    │           且破坏了位置唯一性，注意力分布会紊乱。
    │
    └── Q5.4: 数学本质：
             可学习位置编码假设 **每个位置是一个独立符号**，
             就像为位置也建了一个"词表"。
             外推时遇到的位置相当于 **OOV token**，
             而 NLP 的 OOV 问题在位置维度再次上演。
             归根结底：缺乏归纳偏置（无单调性、无周期性、无函数关系）。
```

**关键洞察**：可学习绝对位置编码就是一个位置词典。它能对每个位置死记硬背，却没办法把位置间的"距离"抽象成规律。因此，当遇到词典里没有的条目，它无法像人类用公式推演那样给出合理答案——这是强记与理解的根本性鸿沟。

### 2. Sinusoidal 绝对位置编码

```
[根] Sinusoidal 位置编码：如何用公式生成无限长的位置信号？
│
├── Q1: 具体公式是什么？逐元素怎么算？
│   ├── Q1.1: 一张表：给定 pos 和维度 i，PE 每个元素的值
│   │        PE(pos, 2i)   = sin( pos / 10000^(2i / d_model) )
│   │        PE(pos, 2i+1) = cos( pos / 10000^(2i / d_model) )
│   │        其中 i = 0, 1, 2, ..., d_model/2 - 1
│   │
│   ├── Q1.2: 为什么同时使用 sin 和 cos 成对出现？
│   │        A: 构成一个二维旋转基底。对于频率 ω_i，
│   │           第 2i 和 2i+1 维组成一个 2D 平面上的向量：
│   │           [sin(pos·ω_i), cos(pos·ω_i)]
│   │           这等价于在二维平面上旋转角度 pos·ω_i 的向量。
│   │           只用 sin 或只用 cos 会丢失相位信息，
│   │           无法区分 pos 和 -pos 的对称性（尽管位置非负，但相对距离可正可负）。
│   │
│   └── Q1.3: 频率 ω_i 的具体取值是怎样的？
│            ω_i = 1 / (10000^(2i / d_model))
│            - 当 i=0（第一对维度）：ω_0 = 1，波长 = 2π ≈ 6.28
│              这是最高频，对相邻位置变化剧烈
│            - 当 i = d_model/2 - 1（最后一对）：ω_max ≈ 1/10000，波长 ≈ 2π×10000
│              这是最低频，变化极慢，允许极远距离的区分
│            不同维度形成几何级数分布的多尺度表示。
│
├── Q2: 为什么这个公式被认为能编码"相对距离"？
│   ├── Q2.1: 点积性质怎么推导？
│   │        设两个位置 pos 和 pos+k。
│   │        PE(pos) 在第 2i,2i+1 维的子向量：
│   │          u = [sin(ω_i·pos), cos(ω_i·pos)]
│   │        PE(pos+k) 对应子向量：
│   │          v = [sin(ω_i·(pos+k)), cos(ω_i·(pos+k))]
│   │        该子空间的点积：
│   │          u·v = sin(ω_i pos) sin(ω_i(pos+k)) + cos(ω_i pos) cos(ω_i(pos+k))
│   │              = cos(ω_i pos - ω_i(pos+k))
│   │              = cos(ω_i k)             ← 仅依赖距离 k！
│   │        整体点积 PE(pos)·PE(pos+k) = Σ_{i} cos(ω_i k)
│   │        这是一个只关于 k 的函数，不含绝对位置 pos。
│   │
│   ├── Q2.2: 这个性质为什么重要？
│   │        A: 模型通过点积 Σ cos(ω_i k) 就可以"感知"距离 k。
│   │           理论上，即使没训练过的绝对位置，只要 k 出现过，
│   │           点积值就是熟悉的。这是其外推优于可学习位置的根本原因。
│   │
│   └── Q2.3: 如果点积只依赖相对距离，那绝对位置信息去哪了？
│            A: 在后续与 W_q, W_k 相乘后，可能引入绝对位置项。
│               纯粹的 PE 点积不含绝对信息，但通过线性层的混合，
│               模型可以恢复一定程度的绝对位置感知。
│               原始论文认为模型可能利用绝对位置来预测下一个词，
│               但实际上相对位置的影响更强。
│
├── Q3: 为什么选择 10000 这个底数？有什么特殊？
│   ├── Q3.1: 数值范围和波长分布
│   │        A: 10000^(2i/d_model) 使得最高频波长约 6.28，最低频约 62831。
│   │           覆盖了从相邻位置到几十万位置的宽谱范围。
│   │           这个选择是经验性的：足够多的频率覆盖大多数训练长度。
│   │
│   ├── Q3.2: 如果底数改为 100 或 1000000 会怎样？
│   │        - 底数越小（如 100）：低频部分波长缩短，远距离位置间
│   │          重复性增加，可能混淆长距离关系。
│   │        - 底数越大（如 1000000）：低频波长极长，短距离时所有频率
│   │          都接近线性变化，位置区分度下降。
│   │        10000 提供了合适的"对数分布"频谱。
│   │
│   └── Q3.3: 这和二进制编码有类比吗？
│            A: 有。不同频率的 sin/cos 可以类比二进制码的不同比特位：
│               高频 → 低位，变化快；低频 → 高位，变化慢。
│               组合起来可以唯一区分每个位置。
│
├── Q4: 训练时 PE 是固定的还是可学习的？
│   ├── Q4.1: 论文中的设定
│   │        A: 固定的，不参与梯度更新。直接作为常量加进去。
│   │
│   ├── Q4.2: 固定有什么好处？
│   │        - 参数零增长，不会过拟合位置
│   │        - 理论上可以自然处理长于训练集的序列（利用公式计算）
│   │        - 实现极简
│   │
│   ├── Q4.3: 能设计成可学习吗？
│   │        A: 可以，但那就变成了可学习绝对位置编码，
│   │           失去了公式外推的可能和归纳偏置。
│   │
│   └── Q4.4: 既然公式固定，位置信息如何适配不同任务？
│            A: 模型通过训练调整 W_q, W_k, W_v 来学会如何"解读"这些固定信号。
│               相当于接收固定的位置编码，但学习如何利用它。
│
├── Q5: 为什么说它的外推能力"有限"？（深层分析）
│   ├── Q5.1: 不是说点积只依赖距离 k 吗？为何外推还会退化？
│   │        A: 因为训练时，模型实际使用的是：
│   │           (W_q (X+PE)) · (W_k (X+PE))
│   │           纯 PE 点积性质只在未混合时成立，混入 W_q, W_k 后，
│   │           展开会包含交叉项 X·PE 和 PE·X，这些项依赖于绝对位置。
│   │           而且 W_q, W_k 在训练时只见过 0~L-1 范围的 PE 值分布。
│   │
│   ├── Q5.2: 深层模型中的"相频干涉"是什么？
│   │        A: 对于超过训练范围的位置，高频维度（如 i=0,1）的 sin/cos
│   │           值可能进入训练时从未出现的"相位组合"区域。
│   │           例如：在 0~L-1 内，某些相对距离 k 对应的总点积 Σ cos(ω_i k)
│   │           具有特定数值范围，但当绝对位置变大后，尽管 k 相同，
│   │           交叉项 X·PE 的分布改变，导致注意力分数偏离习惯的均值/方差。
│   │           这种"偏移"在浅层可能微弱，经过多层堆叠后放大，造成退化。
│   │
│   ├── Q5.3: 具体观察到的退化现象？
│   │        A: 长度超出约 1.5~2 倍训练长度后，PPL 开始显著升高，
│   │           模型生成重复、不连贯的文本。
│   │           退化是渐进式的，不像可学习 PE 那样突然崩溃。
│   │
│   └── Q5.4: 能否通过微调少量长数据恢复？
│            A: 部分可以。少量长文本微调后，模型能缓慢适应更长的范围，
│               但无法像 RPE 类方法那样零次泛化。
│
└── Q6: Sinusoidal 在现代 LLM 中的地位如何？
    ├── Q6.1: 为什么后来被 RoPE 和 ALiBi 取代了？
    │        A: RoPE 继承了正弦的旋转思想但更彻底地融入注意力机制，
    │           且通过 NTK 缩放达到极强外推；
    │           ALiBi 简单而外推极强。
    │           Sinusoidal 在绝大模型场景下外推不够好，训练/推理实现也不如二者优雅。
    │
    ├── Q6.2: 历史贡献是什么？
    │        A: 它首次证明了位置编码不需要学习也能工作，
    │           并展示了用三角函数编码相对距离的思想，
    │           直接启发了 RoPE 的旋转表示。
    │
    └── Q6.3: 还有模型在用吗？
             A: 原始 Transformer、T5 等早期模型使用。
                部分语音模型、时间序列模型仍用，因连续性好。
                在 NLP 大模型领域已较少作为默认选择。
```

### 3. ALiBi (Attention with Linear Biases)

```
[根] ALiBi：如何用一个极简的减法替代所有位置编码？
│
├── Q1: ALiBi 到底改了什么？完整公式是什么？
│   ├── Q1.1: 标准 Attention 计算哪一步被修改了？
│   │        A: 只改一步——在算完 QK^T 之后的原始注意力分数上，
│   │           加上一个静态的、不可学习的偏置矩阵。
│   │
│   ├── Q1.2: 完整的单头注意力公式
│   │        S = Q @ K^T / sqrt(d_head)          ← 标准分数
│   │        B = 构造偏置矩阵 (形状 [T, T])        ← 位置信号
│   │        S' = S + B                          ← 注入位置
│   │        P = softmax(S')                     ← 如常
│   │        O = P @ V                            ← 如常
│   │
│   └── Q1.3: 偏置矩阵 B 具体长什么样？
│            B[i, j] = -m * |i - j|
│            其中 m > 0 是该注意力头的固定斜率。
│            B 是一个 Toeplitz 矩阵，每个对角线上值相同，离对角线越远越小（绝对值越大）。
│
├── Q2: 这个偏置矩阵的含义是什么？
│   ├── Q2.1: 为什么是负号？
│   │        A: 减去正值等于降低远距离 token 的注意力分数。
│   │           距离越远，减得越多，Softmax 后的权重就越小。
│   │           它直接强制执行"局部性先验"：近邻远比远邻重要。
│   │
│   ├── Q2.2: 为什么用线性衰减 |i-j| 而不用指数衰减或其他？
│   │        A: 线性衰减保证了极远距离的惩罚依然可控，
│   │           不会像指数衰减那样把长程交互直接压死。
│   │           而且线性函数的参数（斜率）在小范围内拟合后，
│   │           可以完美的数学外推到任意距离——这是关键。
│   │
│   └── Q2.3: 那每个头的斜率 m 怎么选？
│            A: m 不需要训练，是一组预设的几何级数。
│               例如 8 个头：m ∈ {2^{-8}, 2^{-7}, ..., 2^{-1}}
│                       即 {1/256, 1/128, 1/64, ..., 1/2}
│               不同头有不同斜率 → 不同"视野"。
│
├── Q3: 为什么不训练 m？
│   ├── Q3.1: 如果 m 是可学习的会怎样？
│   │        A: 会退化为学习一个每个头各自的衰减强度，
│   │           但在长序列外推时，学习到的 m 是基于训练长度优化的，
│   │           可能不适用于崭新距离。固定 m 则保证外推时数学一致性。
│   │
│   ├── Q3.2: 几何级数分布的 m 有什么好处？
│   │        A: 可以覆盖从极局部（几乎只看 1~2 个邻居）到
│   │           较远程（注意窗口达几十上百 token）的多个尺度。
│   │           级数系数是经验设定的，一般用 2 的幂次。
│   │
│   └── Q3.3: 如果需要更多样化的距离关系怎么办？
│            A: 这是 ALiBi 的一个弱点：它只提供单调衰减的偏置，
│               无法表达"上一句话的末尾比本句中间更相关"等复杂模式。
│               实际中多头提供了一些多样性，但灵活度不如可学习编码。
│
├── Q4: 为什么 ALiBi 的外推极强？数学原理是什么？
│   ├── Q4.1: 线性规律如何保证外推？
│   │        A: 偏置 B[i,j] = -m * |i-j| 是一个关于距离 k 的纯线性函数。
│   │           训练时，对 k ∈ [1, 2047] 的惩罚被模型学会并适应。
│   │           推理时遇到 k=5000，惩罚 = -m*5000 是严格按同样的线性规律延长。
│   │           这在数学上是外推的最理想情形：没有分布偏移。
│   │
│   ├── Q4.2: 与 Sinusoidal 的"相频干涉"对比
│   │        Sinusoidal 的外推受限于深层网络解读新相位的能力；
│   │        ALiBi 的偏置是一个单调函数，不存在相位、周期等复杂变化。
│   │        整条偏置"直线"被透彻学习，延长后仍是同一条直线。
│   │
│   ├── Q4.3: 与可学习绝对位置对比
│   │        可学习 = 无规律查表，外推完全无定义；
│   │        ALiBi = 严格线性规律，任意距离都有定义且一致。
│   │
│   └── Q4.4: 有没有过长的极限情况？
│            A: 理论上，当序列长度极大（如超过 10 万）时，
│               线性衰减会让远距离的注意力对数分数极小（-m*100k），
│               可能造成软max 下溢。但此时也自然使模型极端局部。
│               这在实际应用中通常可接受。
│
├── Q5: ALiBi 的实现细节与代码层面注意点
│   ├── Q5.1: 偏置矩阵如何高效生成？
│   │        可以预计算一个 [max_len, max_len] 的下三角距离矩阵，
│   │        取出对应序列长度的切片，乘以头对应的 -m。
│   │        实际实施时通常不构建完整矩阵，而是用广播向量。
│   │
│   ├── Q5.2: 与 Causal Mask 的关系？
│   │        Causal Mask 是上三角 -inf，ALiBi 偏置是在此基础上增加下三角的线性衰减。
│   │        两者叠加，同时实现了因果遮挡和距离衰减。
│   │
│   ├── Q5.3: 训练时是否需要为 ALiBi 调整学习率？
│   │        A: 一般不需要。不过由于注意力分布被约束得更局部，
│   │           部分任务可能需要更长的预热步数。
│   │
│   └── Q5.4: ALiBi 可以和其他位置编码混合使用吗？
│            A: 可以，但不必要。通常单独使用即足够，
│               混合可能引入冗余或冲突。但如果外部需要旋转编码，
│               可叠加，但失去了纯粹 ALiBi 的简洁美。
│
├── Q6: ALiBi 的弱点与 trade-off
│   ├── Q6.1: 无法表达非单调或任务特定的位置模式
│   │        A: 比如在阅读理解中，问题-文章之间的交互
│   │           可能不是随单调距离衰减的。ALiBi 强制单调衰减，
│   │           损失了一些灵活性。
│   │
│   ├── Q6.2: 不包含绝对位置信息，对某些任务不利
│   │        A: 某些需要精确计数或绝对序列索引的任务，
│   │           ALiBi 不如可学习绝对位置或 RoPE。
│   │           （语言模型中这类需求通常较弱，所以影响有限。）
│   │
│   └── Q6.3: 为什么没有在所有 LLM 中采用？
│            A: 尽管外推极强，但在同等训练长度内，
│                RoPE 通常有稍好的 PPL 和下游任务性能。
│                且 RoPE 通过 NTK/YaRN 已大幅改善外推，
│                主流模型更愿意用性能更好且外推可调的方案。
│
└── Q7: ALiBi 的历史地位和影响
    ├── Q7.1: 它从哪里来？
    │        A: 2021 年论文《Train Short, Test Long: Attention with Linear Biases
    │           Enables Input Length Extrapolation》。
    │           动机是找到一种不需要任何训练就天然外推的方案。
    │
    ├── Q7.2: 对后续研究的启发？
    │        A: 证明了位置信号完全可以通过修改注意力分数而非输入向量来实现。
    │           并且极简的线性先验可以带来极强外推。
    │           启发了更多"偏置式"相对位置方案的设计。
    │
    └── Q7.3: 哪些知名模型使用了它？
             A: Bloom、MPT 等系列大模型使用 ALiBi 以获取长文本能力。
```

### 4. RoPE (Rotary Position Embedding)

```
[根] RoPE：怎么用旋转优雅地融合位置与内容？
│
├── [第1层：高维旋转]
│   Q1: RoPE 改变了什么？
│       A: 不直接加位置向量，而是对 Q 和 K 向量分别进行"位置相关的旋转"。
│   Q2: 怎么旋转的？具体公式？
│       A: 将 d_head 维向量每两个相邻维度视为一个 2D 平面（共 d_head/2 个平面）。
│          为每个平面分配一个旋转频率 θ_i = 10000^(-2i/d_head)。
│          对位置为 m 的 token，其 Q/K 在该平面的二维向量旋转角度 m·θ_i。
│          数学形式：RoPE(Q,m) = apply_rotary_emb(Q, m)。
│          点积结果只依赖相对位置 m-n：
│          (RoPE_Q_m) · (RoPE_K_n) = Σ Re[ q_i * conj(k_i) * e^{i (m-n)θ_i} ]
│
├── [第2层：为什么如此设计？]
│   Q3: 为什么用旋转而不是直接加？
│       A: 直接加会把位置和内容强行加在一起，互相干扰。
│          旋转是乘法作用在复数模上，它巧妙地"调制"了 Q·K 的内积，
│          使得内积成为"内容相关性"与"位置距离"的混合体，且远离原点时不会爆炸。
│   Q4: 为什么说 RoPE 是"相对位置编码"？
│       A: 因为旋转后的点积只依赖相对距离 m-n。
│          不同绝对位置 (3,5) 和 (3000,3002) 旋转后的点积相等。
│
├── [第3层：训练时学到了什么？]
│   Q5: 模型在训练中怎么利用 RoPE？
│       A: 模型在 0~2048 范围内学习到了：
│          - 短距离（m-n 小）：旋转角度差异小，内积主要由内容决定。
│          - 长距离（m-n 大）：旋转角度差异大，内积被旋转调低，注意力自然稀疏。
│   Q6: θ_i 的取值有什么讲究？
│       A: 高频维度（i 小，θ 大）旋转快，编码局部精细位置；
│          低频维度（i 大，θ 小）旋转慢，编码全局位置。
│          这形成了多尺度的位置表示。
│
└── [第4层：外推的崩溃与救赎]
    Q7: RoPE 既然也是相对位置，为什么外推不如 ALiBi？
         A: 因为 ALiBi 的"距离衰减"是生硬且固定的线性偏置，
            RoPE 的衰减是通过旋转角度"学出来的"，它依赖 θ 的具体数值。
            训练时，长期依赖的距离（比如 1024）对应的最优旋转角已经
            被 W_q, W_k 适应。超出此范围的角度，模型不知道如何处理。
    Q8: 具体崩溃机理？
         A: 当距离 > L_train 时，低频维度的旋转角 (m-n)θ 进入
            训练时从未达到的弧度区域。
            原本应产生高注意力的某些相对距离，可能因为旋转了接近 π
            而点积反号，注意力分布变为均匀噪声（熵崩溃）。
    Q9: NTK/ YaRN 如何手术修复？
         A: 核心思想是"把大距离伪装成小距离"。
            不是修改 m 和 n 的值，而是修改 θ_i（即旋转速度）。
            方法：把基频 10000 放大到 10000 * scale^(d/(d-2))。
            这让低频维度转得更慢，换算后使得 m=5000 时的角度
            近似于改造前 m≈2000 的角度，模型感觉"仍在舒适区"。
    Q10: 效果如何？
         A: 稍微损失一点极短距离的分辨率，换取数十倍的外推长度。
```

**一句话总结**：RoPE 把位置编码成旋转，天然相对但需要学角度，外推时角度超界导致失焦，NTK 手术把表调慢让它回归清晰。

## 终极总结

### 位置编码的"学习"含义辨析

| 含义 | 代表方案 | 解释 |
|------|----------|------|
| 参数学习 | 可学习绝对位置 | 通过梯度下降直接优化位置向量 |
| 模型适应 | Sinusoidal、RoPE | 固定公式，模型学会解读信号 |
| 人为注入先验 | ALiBi | 不需要学习，直接规定规律 |

### 四种方案对比

| 方案 | 核心机制 | 外推崩溃的根因 | 修复策略 | "学"的含义 |
|------|----------|----------------|----------|------------|
| 可学习 | 查表背位置 | 查不到 | 无（只能截断） | 参数学习 |
| Sinusoidal | 三角函数公式 | 新相位的解读失效 | 实践中较少单独修复 | 模型适应 |
| ALiBi | 线性距离减法 | 几乎不崩 | 天然免修 | 人为注入先验 |
| RoPE | 复数旋转 | 旋转角超出训练范围导致熵崩塌 | NTK/YaRN 缩放旋转频率 | 模型适应 |

### 位置编码的进化史

```
强记位置 → 理解相对距离函数 → 解耦位置与语义
  ↓              ↓                    ↓
可学习PE    Sinusoidal/RoPE      RoPE/ALiBi
```

**终极浓缩**：位置编码的"学习"，不是学位置本身的内容，而是学"位置在注意力机制中应该如何表现的规律"。

- 可学习绝对位置：死记硬背每个位置的角色
- 公式化绝对位置：用固定波形提供距离函数，让模型解读
- 相对偏置：直接注入"越远越不重要"的先验
- 旋转编码：让内积变成相对距离的天然载体，实现位置与内容的解耦

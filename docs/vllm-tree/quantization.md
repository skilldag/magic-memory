## 表层问题

**输入**：权重 FP32
**输出**：权重 INT8
**问题**：如何减少显存和计算？

## 第二层推导：量化方案

### 2.1 INT8 量化

```
[INT8]
    │
    ├── 公式：x_q = round(x_fp32 / scale)
    │
    ├── scale 计算：
    │       ├── per-tensor: 一个scale
    │       └── per-channel: 多个scale
    │
    └── 反量化：x_fp32 = x_q * scale
```

### 2.2 INT4 量化

```
[INT4]
    │
    ├── 压缩率：8x
    │
    ├── 问题：精度损失更大
    │       解决：更精细的scale
    │
    └── 存储：两个4-bit打包
```

### 2.3 AWQ / GPTQ

```
[AWQ]
    │
    ├── 问题：权重分布不均
    │       解决：activation感知
    │
    └── 核心：保护重要权重

[GPTQ]
    │
    ├── 问题：如何最小化误差
    │       解决： Hessian近似
    │
    └── 核心：逐通道优化
```

## 第三层推导：校准

### 3.1 校准方法

```
[校准]
    │
    ├── 动态量化
    │       └── 实时计算 scale
    │
    ├── 静态量化
    │       └── 离线计算 scale
    │
    └── 校准数据
            └── 代表性采样
```

### 3.2 误差补偿

```
[Error Compensation]
    │
    ├── 问题：量化误差累积
    │       解决：后训练微调
    │
    └── 问题：特定层误差大
            解决：混合精度
```

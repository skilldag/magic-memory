## 表层问题

**输入**：用户请求 "Write a story about a dragon"
**输出**：生成的文本
**问题**：请求如何分阶段处理？

---

## 第一层推导：为什么需要分阶段？

```
[请求 → 生成]
      │
  ┌───┴───┐
  │       │
[问题1] [问题2]
  │       │
  ▼       ▼
[如何处理] [如何优化]
  │       │
  └───┬───┘
      │
  [Prefill/Decode]
```

### 问题1：为什么需要分阶段？

**问题**：为什么不能一次性处理整个请求？

**原因分析**：
```
1. 请求包含 prompt（长）和生成（逐token）
2. prompt 只需要计算一次
3. 生成需要逐token计算
4. 混在一起效率低
```

**解决**：分阶段处理

### 问题2：如何优化？

**问题**：如何提高吞吐量？

**原因分析**：
```
1. Prefill 阶段：计算密集，可以并行
2. Decode 阶段：内存密集，串行
3. 需要不同的优化策略
```

**解决**：Prefill/Decode 分离

---

## 第二层推导：Prefill 阶段

### 2.1 Prefill 详细推导

```
[Prefill]
    │
    ├── 问题：如何处理 prompt？
    │       解决：一次性计算所有 token
    │
    ├── 问题：如何利用并行？
    │       解决：矩阵并行计算
    │
    └── 问题：如何存储 KV？
            解决：生成 KV Cache
```

**为什么叫 Prefill？**
- 预填充（pre-fill）
- 先填充 KV Cache

### 2.2 Decode 详细推导

```
[Decode]
    │
    ├── 问题：如何生成下一个 token？
    │       解决：逐 token 生成
    │
    ├── 问题：如何利用 KV Cache？
    │       解决：复用 Prefill 的 KV
    │
    └── 问题：如何优化？
            解决：批处理
```

**为什么叫 Decode？**
- 解码（decode）
- 逐 token 解码生成

---

## 第三层推导：优化策略

### 3.1 Prefill 优化

**问题**：Prefill 如何优化？

**解决**：
```
1. FlashAttention：加速 Attention
2. 并行计算：充分利用 GPU
3. KV Cache：避免重复计算
```

### 3.2 Decode 优化

**问题**：Decode 如何优化？

**解决**：
```
1. 批处理：多个请求一起
2. KV Cache 复用：减少计算
3. Speculative Decoding：推测加速
```

## 完整推导树

```
Prefill/Decode
    │
    ├── [Prefill]
    │       ├── 问题：为什么需要预填充？
    │       │       解决：生成 KV Cache
    │       │
    │       ├── 问题：如何处理 prompt？
    │       │       解决：一次性计算
    │       │
    │       └── 问题：如何优化？
    │               解决：并行计算
    │
    └── [Decode]
            ├── 问题：为什么需要逐 token？
            │       解决：自回归
            │
            ├── 问题：如何生成？
            │       解决：逐 token 生成
            │
            └── 问题：如何优化？
                    解决：批处理 + KV 复用
```

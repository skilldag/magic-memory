## 第一层推导：为什么需要队列？

```
[请求 → 队列]
      │
  ┌───┴───┐
  │       │
[问题1] [问题2] [问题3]
  │       │       │
  ▼       ▼       ▼
[如何存储] [如何排序] [如何调度]
  │       │       │
  └───┬───┴───┘
      │
  [Request Queue]
```

### 问题1：如何存储？

**问题**：如何存储待处理的请求？

**原因分析**：
```
1. 请求可能比 GPU 容量多
2. 需要排队等待
3. 需要存储请求信息
4. 需要跟踪状态
```

**解决**：队列

### 问题2：如何排序？

**问题**：请求按什么顺序处理？

**原因分析**：
```
1. 先来先服务（FCFS）
2. 优先级调度
3. 需要排序策略
```

**解决**：排序策略

### 问题3：如何调度？

**问题**：如何从队列中取出请求？

**原因分析**：
```
1. 队列满了怎么办？
2. 如何选择下一个请求？
3. 如何处理优先级？
```

**解决**：调度器

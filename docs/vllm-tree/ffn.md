### 问题1：为什么需要非线性？

**问题**：为什么不能只有 Attention？

**原因分析**：
```
1. Attention 是线性变换
2. 线性变换表达能力有限
3. 需要非线性变换
4. 需要增加模型容量
```

**对比**：

| 变换 | 表达能力 | 计算量 | 问题 |
|------|----------|--------|------|
| **线性** | 有限 | 小 | 表达能力不足 |
| **非线性** | 强 | 大 | **需要非线性** |

**结论**：需要 FFN 来增加非线性表达能力

### 问题2：如何实现？

**问题**：如何实现非线性变换？

**解决**：
```
1. 两层全连接
2. 中间加激活函数
3. 输入→FC1→激活→FC2→输出
```

### 问题3：如何优化？

**问题**：如何优化 FFN？

**解决**：
```
1. SwiGLU 激活函数
2. 门控机制
3. 提高表达能力
```

## 第三层推导：实现细节

### 3.1 两层 FC

**问题**：为什么两层？

**解决**：
```
1. 第一层：升维（hidden_dim → 4*hidden_dim）
2. 激活函数
3. 第二层：降维（4*hidden_dim → hidden_dim）
```

### 3.2 激活函数

**问题**：如何选择激活函数？

**解决**：
```
- ReLU: 简单，但可能死神经元
- GELU: 平滑，性能好
- SwiGLU: 门控，性能最好
```

## 表层问题

**输入**：已生成 tokens [t1, t2, t3]
**输出**：新 token t4
**问题**：生成过程如何一步步推进？

## 第一层推导：为什么需要 Decode Step？

```
[生成 → 下一步]
      │
  ┌───┴───┐
  │       │
[问题1] [问题2]
  │       │
  ▼       ▼
[如何推进] [何时停止]
  │       │
  └───┬───┘
      │
  [Decode Step]
```

### 问题1：如何推进？

**问题**：如何从已生成 token 生成下一个？

**原因分析**：
```
1. 自回归：下一个 token 依赖之前的所有 token
2. 需要逐步生成
3. 每次生成一个 token
4. 需要循环
```

**解决**：Decode Step（解码步）

### 问题2：何时停止？

**问题**：如何判断生成完成？

**原因分析**：
```
1. 生成可能无限继续
2. 需要停止条件
3. 可能是终止 token
4. 可能是最大长度
```

**解决**：终止检查

## 第二层推导：Decode Step 流程

### 2.1 Decode Step 详细推导

```
[Decode Step]
    │
    ├── 问题：如何计算？
    │       解决：前向传播
    │
    ├── 问题：如何采样？
    │       解决：采样器
    │
    └── 问题：如何判断？
            解决：终止检查
```

**为什么叫 Decode Step？**
- 解码（decode）的步骤
- 每次生成一个 token

### 2.2 终止检查详细推导

```
[终止检查]
    │
    ├── 问题：如何判断？
    │       ├── token == EOS
    │       ├── 长度 >= max_length
    │       └── 用户停止
    │
    └── 问题：如何处理？
            解决：返回结果
```

## 第三层推导：优化

### 3.1 Decode Step 优化

**问题**：如何加速 Decode Step？

**解决**：
```
1. KV Cache 复用：减少计算
2. 批处理：多个请求一起
3. Speculative Decoding：推测加速
```

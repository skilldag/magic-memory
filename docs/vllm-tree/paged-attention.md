## 第二层推导：核心技术

### 2.1 Block (块)

```
[块]
    │
    ├── 问题：块多大？
    │       解决：固定大小 (如 16 tokens)
    │
    ├── 块类型：
    │       ├── 物理块 (physical)
    │       └── 虚拟块 (logical)
    │
    └── 问题：如何分配？
            先分配物理块
```

### 2.2 Block Table (块表)

```
[块表映射]
    │
    ├── 问题：虚拟块→物理块？
    │       解决：指针映射
    │
    ├── 结构：
    │       [block_indices] → 物理块ID
    │       [block_valid] → 是否有效
    │
    └── 问题：复制/移动？
            改指针，不搬数据
```

### 2.3 引用计数

```
[引用计数]
    │
    ├── 问题：多请求共享块？
    │       解决：引用计数
    │
    ├── 问题：何时释放？
    │       计数=0时释放
    │
    └── 问题：写时复制？
            计数>1时复制
```

## 第三层推导：实现细节

### 3.1 分配流程

```
1. 请求到来，分配 N 个虚拟块
2. 首次写入，分配物理块
3. 更新 block table
4. 引用计数 = 1
```

### 3.2 共享流程

```
1. 请求 Bcopy 请求 A 的 prefix
2. 复制 block table 指针
3. 引用计数 ++
4. 写时复制
```

### 3.3 释放流程

```
1. 请求完成
2. 引用计数 --
3. 计数 == 0？释放物理块
4. 释放虚拟块
```

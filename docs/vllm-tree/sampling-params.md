## 第一层推导：为什么需要采样参数？

```
[用户需求 → 采样参数]
      │
  ┌───┴───┐
  │       │
[问题1] [问题2] [问题3]
  │       │       │
  ▼       ▼       ▼
[随机性] [候选数] [分布]
  │       │       │
  └───┬───┴───┘
      │
  [Sampling Params]
```

### 问题1：如何控制随机性？

**问题**：如何控制生成的确定性和随机性？

**原因分析**：
```
1. 用户有时需要确定性输出
2. 有时需要随机性
3. 需要一个参数控制
```

**解决**：Temperature（温度）

### 问题2：如何控制候选数？

**问题**：如何限制候选 token 数量？

**原因分析**：
```
1. 词表太大（5万+）
2. 不需要考虑所有 token
3. 只考虑前 K 个
```

**解决**：Top-K

### 问题3：如何控制分布？

**问题**：如何截断概率分布？

**原因分析**：
```
1. 概率分布太长尾
2. 只考虑累积概率达到 P 的 token
3. 需要动态调整候选数
```

**解决**：Top-P（Nucleus）

## 第二层推导：参数详解

### 2.1 Temperature 详细推导

```
[Temperature]
    │
    ├── 问题：如何控制随机性？
    │       ├── T > 1: 更随机
    │       │       解决：降低峰值
    │       ├── T = 1: 标准
    │       └── T < 1: 更确定
    │               解决：放大峰值
    │
    └── 问题：如何应用？
            解决：除以温度
```

**为什么叫 Temperature？**
- 温度，控制"热度"
- 高温=随机，低温=确定

### 2.2 Top-K 详细推导

```
[Top-K]
    │
    ├── 问题：如何选择候选？
    │       解决：保留前 K 个
    │
    ├── 问题：K 如何选择？
    │       解决：经验值（40-50）
    │
    └── 问题：如何归一化？
            解决：重新归一化
```

**为什么叫 Top-K？**
- 保留前 K 个
- K 是超参数

### 2.3 Top-P 详细推导

```
[Top-P]
    │
    ├── 问题：如何截断？
    │       解决：累积概率 ≥ P
    │
    ├── 问题：P 如何选择？
    │       解决：经验值（0.9）
    │
    └── 问题：如何动态调整？
            解决：根据分布形状
```

**为什么叫 Top-P？**
- Nucleus sampling
- P 是累积概率阈值

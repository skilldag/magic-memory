### 问题1：为什么需要多头？

**问题**：为什么不能只用一个头？

**原因分析**：
```
1. 单头只能关注一种关系
2. 不同的 token 有不同的关系
3. 语法、语义、位置关系不同
4. 需要多个头关注不同方面
```

**对比**：

| 方式 | 优点 | 缺点 | 问题 |
|------|------|------|------|
| **单头** | 简单 | 只能关注一种关系 | 信息有限 |
| **多头** | 关注多种关系 | 计算量大 | **需要多头** |

**结论**：需要多头来关注不同类型的关系

### 问题2：如何实现？

**问题**：如何实现多头？

**解决**：
```
1. 将 Q, K, V 分割成 h 个头
2. 每个头独立计算 Attention
3. 每个头关注不同方面
4. 最后拼接所有头
```

### 问题3：如何合并？

**问题**：多头输出如何合并？

**解决**：
```
1. 拼接所有头的输出
2. 线性投影
3. 得到最终输出
```

## 第二层推导：多头结构

### 2.1 Multi-Head 详细推导

```
[Multi-Head]
    │
    ├── 问题：如何分割？
    │       解决：按头维度分割
    │
    ├── 问题：如何计算？
    │       解决：每个头独立计算
    │
    └── 问题：如何合并？
            解决：拼接 + 投影
```

**为什么叫 Multi-Head？**
- 多个头（multi-head）
- 每个头关注不同方面

### 2.2 头数选择

**问题**：头数如何选择？

**原因分析**：
```
1. 头数越多，计算量越大
2. 头数太少，信息不足
3. 需要平衡
```

**解决**：
```
- LLaMA: 32 头
- GPT: 12 头
- BERT: 12 头
```

## 第三层推导：实现细节

### 3.1 头分割

**问题**：如何分割 Q, K, V？

**解决**：
```
1. hidden_dim 分成 h 个头
2. 每个头的维度 = hidden_dim / h
3. Q, K, V 分别分割
```

### 3.2 头计算

**问题**：每个头如何计算？

**解决**：
```
1. 每个头独立计算 Attention
2. 使用相同的权重
3. 并行计算
```

### 3.3 头合并

**问题**：如何合并多头输出？

**解决**：
```
1. 拼接所有头的输出
2. 线性投影回 hidden_dim
3. 得到最终输出
```

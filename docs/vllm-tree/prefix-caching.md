# Prefix Caching 深层推导

> 从"相同前缀如何复用"往下推，推到不能再分

---

## 表层问题

**输入**：两个请求 ["Hello, how are", "Hello, how can"]
**输出**：相同前缀只计算一次
**问题**：相同前缀的重复计算能否避免？

## 第二层推导：Hash

### 2.1 哈希函数

```
[Hash]
    │
    ├── 问题：哈希碰撞？
    │       解决：加密哈希
    │
    ├── 常用：
    │       ├── MD5 (128-bit)
    │       ├── SHA-256
    │       └── xxHash (快速)
    │
    └── 问题：部分前缀？
            解决：多哈希
```

### 2.2 哈希内容

```
[What to Hash]
    │
    ├── 问题：什么需要哈希？
    │       解决：prompt + 采样参数
    │
    ├── 需要包含：
    │       ├── token 序列
    │       ├── temperature
    │       └── top_p / top_k
    │
    └── 相同才算 cache hit
```

## 第三层推导：存储与驱逐

### 3.1 存储结构

```
[Storage]
    │
    ├── Key: hash(prefix)
    │       │
    └── Value:
            ├── KV cache
            ├── 位置信息
            └── 引用计数
```

### 3.2 驱逐策略

```
[Eviction]
    │
    ├── LRU
    │       └── 最近最少使用
    │
    ├── LFU
    │       └── 最少使用
    │
    └── Size-based
            └── 优先驱逐大的
```

### 3.3 共享方式

```
[Sharing]
    │
    ├── 只读共享
    │       │
    └── 写时复制
            └── 引用计数
```

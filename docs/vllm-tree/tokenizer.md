### 问题1：为什么需要分词？

**问题**：文本不能直接输入模型，为什么？

**原因分析**：
```
1. 模型只能处理数字，不能处理文本
2. 需要转换成数值序列
3. 但如何切分文本？
```

**三种切分方式对比**：

| 切分方式 | 优点 | 缺点 | 问题 |
|----------|------|------|------|
| **字符级** | 词表小（256） | 序列太长 | 计算量大 |
| **词级** | 序列短 | 词表太大（百万级） | 未登录词多 |
| **子词级** | 折中方案 | 需要算法 | **需要分词算法** |

**结论**：需要分词算法来平衡词表大小和序列长度

### 问题2：为什么有多种分词算法？

**问题**：为什么有 BPE/WordPiece/SentencePiece？

**原因分析**：
```
不同算法解决不同问题：
- BPE：基于统计，简单高效
- WordPiece：基于似然，训练数据友好
- SentencePiece：端到端，统一处理
```

## 第二层推导：分词模式

### 2.1 BPE (Byte-Pair Encoding)

**问题**：如何合并高频字节对？

**原因**：
- 字符级分词序列太长
- 需要合并成更大的单元
- 但合并哪些？如何合并？

**解决**：
```
1. 统计所有相邻字节对的频次
2. 从高频到低频合并
3. 重复直到达到词表大小
```

**为什么叫 Byte-Pair？**
- 基于字节对（byte pair）的合并
- 每次合并两个相邻字节

**示例**：
```
原始: h e l l o   w o r l d
     ↓ 合并 'll'
h e ll o   w o r l d
     ↓ 合并 'lo'
h e ll o   w o r l d
     ↓ 合并 'll' + 'o' = 'llo'
h e llo   w o r l d
```

### 2.2 WordPiece

**问题**：如何选择子词让训练数据概率最大？

**原因**：
- BPE 只看频次，不考虑训练数据
- WordPiece 考虑训练数据的似然

**解决**：
```
1. 计算每个子词的似然贡献
2. 选择使训练数据概率最大的子词
3. 贪心选择
```

**为什么叫 WordPiece？**
- 把词拆成"piece"（片段）
- 每个片段都是词的一部分

### 2.3 SentencePiece

**问题**：如何端到端训练分词器？

**原因**：
- BPE/WordPiece 需要预处理
- 空格处理不一致
- 想要统一的端到端方案

**解决**：
```
1. 把空格当作特殊字符
2. 端到端训练
3. 统一处理所有文本
```

**为什么叫 SentencePiece？**
- 把句子拆成"piece"
- 统一处理整个句子

## 第三层推导：实现细节

### 3.1 BPE 实现步骤

```
1. 按字符分割 → 字节序列
2. 统计所有相邻对频次
3. 合并频次最高的对
4. 更新词表
5. 重复直到达到词表大小
```

### 3.2 Byte Fallback

**问题**：未登录词怎么办？

**原因**：
- 词表有限，总有词不在词表中
- 不能直接丢弃

**解决**：
```
1. 拆成字节
2. 每个字节都在词表中（256个字节）
3. 保证任何词都能表示
```

**为什么叫 Byte Fallback？**
- 回退到字节级别
- 保证总能表示

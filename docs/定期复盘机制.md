# 定期复盘机制

> 基于遗忘曲线的 vLLM 知识巩固策略

## 核心理念

学习不是一次性行为，而是需要周期性复习的过程。根据艾宾浩斯遗忘曲线，新学知识的保留率会随时间快速下降。本机制帮助你制定科学的复盘计划，确保 vLLM 知识长期记住。

---

## 一、遗忘曲线原理

### 遗忘规律

| 时间间隔 | 记忆保留率 |
|----------|-----------|
| 20分钟 | 58% |
| 1小时 | 44% |
| 1天 | 33% |
| 2天 | 27% |
| 6天 | 25% |
| 31天 | 21% |

### 复盘时机

关键复盘时间点：
- 第 1 天：当天学习后
- 第 2 天：第 1 次复盘
- 第 4 天：第 2 次复盘
- 第 7 天：第 3 次复盘
- 第 14 天：第 4 次复盘
- 第 30 天：第 5 次复盘
- 第 90 天：第 6 次复盘

## 二、复盘计划

### 每日复盘（5-10 分钟）

**时间**：每天固定时间，如早上或晚上

**内容**：
1. 快速过一遍 0-50 编号和概念（3 分钟）
2. 随机抽取 5 个编号，说出对应概念和核心职责（2 分钟）
3. 复习前一天不熟悉的概念（2 分钟）

**记录**：

```
## 每日复盘 - [日期]

### 快速回忆
- 正确数：___/50
- 用时：___秒

### 抽查结果
- 编号 7：___（正确/错误）
- 编号 23：___（正确/错误）
- 编号 38：___（正确/错误）
- 编号 41：___（正确/错误）
- 编号 49：___（正确/错误）

### 需要加强
- ___
```

### 每周复盘（30 分钟）

**时间**：每周日或周末

**内容**：
1. 完成一套 Level 自测题（15 分钟）
2. 画一条完整的数据流路径（5 分钟）
3. 回答 3 个深度理解问题（10 分钟）

**自测题示例**：

问题 1：从 43 说到 23，完整描述请求处理流程。

问题 2：解释 PagedAttention 如何解决显存碎片化。

问题 3：什么场景下应该启用 Speculative Decoding？

**记录**：

```
## 每周复盘 - 第[N]周

### 本周学习内容
- 完成了___个 Level
- 自测正确率：___%

### 掌握情况
- 熟练：___
- 模糊：___
- 忘记：___

### 问题与改进
- 问题：___
- 改进：___
```

### 每月复盘（1-2 小时）

**时间**：每月最后一天

**内容**：
1. 完成全部 Level 的综合测试（30 分钟）
2. 绘制完整的概念关联图（30 分钟）
3. 解决 2 个场景实战题（30 分钟）

**综合测试题目**：

题目 1：遇到 OOM 错误，列出排查步骤和涉及的概念编号。

题目 2：设计一个高并发 vLLM 服务，列出关键组件和配置。

**记录**：

```

### 整体评估
- Level 1 掌握度：___%
- Level 2 掌握度：___%
- Level 3 掌握度：___%

### 场景实战

- 场景 2：___（通过/不通过）
```

## 三、复盘方法

### 方法一：主动回忆

不要翻看资料，尝试闭眼回忆。

1. 随机选择一个编号
2. 说出对应的锚点图像
3. 说出对应的概念
4. 说出概念的核心职责
5. 说出相关的概念

如果说不出来，记下来，稍后重点复习。

### 方法二：讲解给他人

找一个不懂 vLLM 的人，给他讲解概念。

1. 从最简单的概念讲起
2. 讲概念之间的关系
3. 讲完整的请求流程

讲不通的地方就是你理解不深的地方。

### 方法三：解决问题

不要只回忆概念，要解决问题。

1. 随机选择一个场景
2. 分析涉及哪些概念
3. 提出解决方案

解决问题是检验理解的最终标准。

### 方法四：对比联想

把相似或相关的概念放在一起对比。

1. PagedAttention vs 标准 Attention
2. Prefill vs Decode
3. Continuous Batching vs 静态批处理
4. gRPC vs WebSocket

对比能加深理解。

## 五、不熟练概念处理

### 分类处理

**完全忘记**：编号和概念都对不上号

- 重新记忆锚点图像
- 理解概念的定义
- 联想相关概念

**记忆模糊**：能说对编号和概念，但细节说不清

- 回顾锚点文档
- 回答相关问题
- 在场景中运用

**理解不深**：能说概念，但说不清为什么

- 查看源码
- 阅读官方文档
- 解决实际问题

### 常见困难概念

根据经验，以下概念较难理解：

| 概念 | 难度 | 建议 |
|------|------|------|
| PagedAttention | 高 | 结合 Block Table 和 KVCacheManager 一起理解 |
| Speculative Decoding | 高 | 先理解自回归解码的瓶颈，再理解推测原理 |
| Continuous Batching | 中 | 对比静态批处理，理解动态调整的好处 |
| Distributed | 高 | 先理解单卡推理，再扩展到多卡 |

## 八、形成习惯

### 复盘节奏建议

| 周期 | 时间 | 内容 |
|------|------|------|
| 每日 | 5-10分钟 | 快速回忆 |
| 每周 | 30分钟 | 阶段性测试 |
| 每月 | 1-2小时 | 综合评估 |
| 每季度 | 半天 | 系统性回顾 |

### 坚持技巧

1. 固定时间：每天同一时间复盘，形成习惯
2. 记录可见：用笔记软件记录，看到进度
3. 目标明确：每次复盘有具体目标
4. 奖励机制：完成复盘后给自己小奖励

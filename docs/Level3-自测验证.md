## 答案提示（先思考再查看）

### 30-33：推测解码

**答案 1**：解决自回归解码的等待问题，通过推测多个 token 来减少等待时间
**答案 2**：Draft（起草）→ Verify（验证）→ Accept（接受）
**答案 3**：N-gram 快但质量低，模型 proposer 质量高但慢，常组合使用

### 34-37：批处理与调度

**答案 1**：静态批处理固定批大小和时间，连续批处理动态调整
**答案 2**：管理请求队列，决定何时调度哪个请求，分配资源
**答案 3**：Prefill 优先适合长输入短输出，Decode 优先适合短输入长输出
**答案 4**：两个阶段计算特性不同，可以分别优化（并行 vs 串行）

### 38-39：缓存优化

**答案 1**：相同前缀的请求可以复用 KV Cache，适合系统提示词长的场景
**答案 2**：FIFO 或优先级队列，管理请求的入队和出队
**答案 3**：LRU、LFU 等，vLLM 使用 LRU

### 40-43：服务层

**答案 1**：Engine 是核心引擎，API 是暴露给外部的接口层
**答案 2**：Serving 是完整的服务封装，包含 API、Engine、配置等
**答案 3**：可以用 OpenAI 的 API 调用方式来调用 vLLM

### 44-45：通信协议

**答案 1**：gRPC 更高效，适合内部调用；HTTP 更通用，适合外部调用
**答案 2**：适合需要实时推送的场景，如流式输出

### 46-47：多模型与驱动

**答案 1**：多个 LoRA 适配器，可以动态切换，适用于微调场景
**答案 2**：Driver 是硬件抽象层，vLLM 通过 Driver 与 GPU 交互

### 48-50：高级特性

**答案 1**：通过哈希或前缀树快速查找已缓存的前缀
**答案 2**：显存不足时触发，驱逐最少使用的缓存块
**答案 3**：解决单卡显存不足和推理延迟问题，需要考虑通信开销、数据分片等
